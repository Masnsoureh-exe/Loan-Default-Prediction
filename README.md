# پیش‌بینی ریسک نکول وام (Loan Default Risk Prediction)

پیش‌بینی احتمال نکول (default) متقاضی وام، با استفاده از اطلاعات جمعیت‌ شناختی، مالی و شغلی متقاضی.

## مسئله
با توجه به پروفایل متقاضی (درآمد، سن، شغل، محل سکونت، مالکیت خانه/خودرو، سابقه شغلی و...)، هدف پیش‌بینی متغیر دودویی `Risk_Flag` است؛ یعنی اینکه آیا احتمال دارد متقاضی در بازپرداخت وام نکول کند یا نه.

## رویکرد
- **تحلیل اکتشافی داده (EDA)** — بررسی توزیع ویژگی‌ها، مقادیر گمشده، داده‌های تکراری، و رابطه هر ویژگی دسته‌ای با نرخ نکول (شهر، شغل، استان، مالکیت خودرو/خانه، وضعیت تأهل)
- **بررسی داده‌های پرت** — استفاده از روش IQR روی ویژگی درآمد (هیچ داده پرتی یافت نشد)
- **مهندسی ویژگی (Feature Engineering)**
  - تبدیل ویژگی سن به یک ویژگی دسته‌ای به نام `age_group` (جوان / میانسال / مسن)
  - ساخت ویژگی `job_stability_ratio` (نسبت سابقه شغلی فعلی به کل سابقه کاری)
- **رمزگذاری (Encoding)**
  - رمزگذاری دودویی برای ویژگی‌های دسته‌ای دو-کلاسه (وضعیت تأهل، مالکیت خودرو)
  - رمزگذاری One-Hot برای استان، شغل، و مالکیت خانه
  - رمزگذاری فراوانی (Frequency Encoding) برای شهر (به دلیل تعداد زیاد دسته‌ها)
  - رمزگذاری ترتیبی (Ordinal Encoding) برای ویژگی مهندسی‌شده گروه سنی
- **مدل‌سازی** — مقایسه سه مدل رگرسیون لجستیک، درخت تصمیم، و Random Forest با استفاده از اعتبارسنجی متقاطع Stratified K-Fold، با معیار ارزیابی **ROC-AUC** (به‌جای دقت ساده، به دلیل عدم توازن کلاس‌ها در متغیر هدف)
- **تنظیم هایپرپارامتر** — استفاده از GridSearchCV روی بهترین مدل (Random Forest)
- **ارزیابی نهایی** — علاوه بر ROC-AUC، از معیار **H-Measure** (مقاوم در برابر عدم توازن کلاس‌ها و هزینه نامتقارن خطای طبقه‌بندی) و **منحنی Lift** برای سنجش میزان کارایی مدل در شناسایی متقاضیان پرریسک نسبت به انتخاب تصادفی استفاده شد

## نتیجه
پس از تنظیم هایپرپارامتر، مدل Random Forest بهترین عملکرد را داشت که با معیارهای ROC-AUC، H-Measure، و Lift در ۱۰٪ بالای متقاضیان پرریسک (بر اساس پیش‌بینی مدل) روی داده‌های ارزیابی (validation) سنجیده شد.
